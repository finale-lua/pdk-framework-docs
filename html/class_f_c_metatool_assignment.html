Detailed Description

Class for a metatool key assignment.

Before using an object, the mode (=Finale tool) must be set with SetMode.

Metatool are saved to the keystroke location. Example: a metatool record for the 'A' key are stored at position 65 (ASCII for 'A'), so GetItemNo would return 65.

Tuplets are only supported if PDK_FRAMEWORK_ENTRIES is defined to the compiler.

◆ CreateLinkedObject()

__FCBaseData * FCMetatoolAssignment::CreateLinkedObject ( )

Creates and loads an object that is connected to the metatool keystroke.

The caller is reponsible for deleting the created object from the heap.

Please note that the returned object is different depending on the object type.

  • Articulation metatools will return a FCArticulationDef object.
  • Text Expressions metatools will return a FCTextExpressionDef object.
  • Shape Expression metatools will returh a FCShapeExpressionDef object.
  • Repeat metatools will return a FCTextRepeatDef object.

  • Tuplet metatools will return a FCTuplet object.

    • Staff Style metatools will return a FCStaffStyleDef object.

    Lua-supported.

Returns
NULL if the a linked object type isn't supported or couldn't be loaded. Otherwise, the pointer to the created object. The type of the object varies, depending on the object type.

◆ GetDefID()

CMPER FCMetatoolAssignment::GetDefID ( ) const
inline

Returns the 1-based (or 0-based) definition ID for supported object types.

These types are supported:

  • Expressions (1-based FCTextExpressionDef or FCShapeExpressionDef ID)
  • Articulations (1-based FCArticulationDef ID)
  • Repeats (1-based FCTextRepeatDef ID)
  • Clefs (0-based FCClefDef ID)
  • Staff Styles (1-based FCStaffStyleDef ID)

These types are not supported:

  • Tuplets

Lua-supported (also as a read-only property).

Returns
0 if the mode isn't supported. Otherwise, the 1-based ID.
