Detailed Description

Base class for all data-related classes (that handles Finale data).

This class is abstract and is overridden by child classes. Should not be used directly.

◆ CreateRawDataDump()

FCNumbers * __FCBaseData::CreateRawDataDump ( )

Creates a byte collection with the raw loaded data block for the object (if the object type supports single data blocks). This method is only intended for debug purposes.

The first byte is in the first item of the collection, and so on. The size of the collection (in number of items) is the size of the data (in bytes).

Each byte in the collection is in the 0-255 decimal range.

If no data has been loaded, NULL is returned. Otherise, the created FCNumbers object containing the data dump.

Lua-supported.

◆ DeleteData()

bool __FCBaseData::DeleteData ( )
virtual

Deletes the associated data from Finale's database. Be careful when deleting multiple objects.

◆ RelinkToCurrentView()

bool __FCBaseData::RelinkToCurrentView ( )

Relinks the contents in the current view so that the score and part(s) match.

If the current edit focus is the score, the parts are relinked to the score. If the current edit focus is a part, all parts and score are linked with the values in the current part.

If you process items in a collection to relink them, use eachbackwards to itereate the collection rather than iterating it forwards.

Warning
A Finale plugin has no direct access to linkage in Finale. This function works by deleting the data item and recreating it as a new item. This indirectly clears all linkage. However, that means this function does not know if a data item is unlinkable in Finale. Relink has been tested with all known unlinkable items. If you call it on other items, it will more than likely delete and re-add it successfully. But this is not supported behavior.
One known issue is FCBaseline. When you delete a lyric baseline that is the default baseline for that lyric number, Finale (as of Finale 27.3) deletes all defaults for subsequent lyric numbers. This behavior does not appear to be controllable by plugins. If you relink the first one you may end up relinking them all. You should plan for that in how you deal with relinking FCBaseline instances.

Lua-supported (0.67).

Returns
success if true
