Detailed Description

Class for key signatures. Instances of this class are auto-created by FCMeasure:GetKeySignature and FCCell:GetKeySignature when needed.

Signatures are normally set with SetMajorKey or SetMinorKey. You can also use SetID to set the signature from FCCustomKeyModeDef::GetKeyID. The owner measure/cell must be saved afterwards.

Transpositions should NOT be set in the key for chromatic transpositions.

◆ CalcDiatonicStepsMap()

std::vector< eHarmLev > FCKeySignature::CalcDiatonicStepsMap ( ) const

Returns an array (or table in Lua) specifiying the chromatic step where each diatonic step occurs. The first element is always zero. The elements always increase, and the final element must be less than CalcTotalChromaticSteps. If the array has a number of elements other than 7, it can't easily be represented on a traditional staff, so the chance of a number of elements other than 7 is vanishingly small. But Finale allows it.

Predefined major keys always return: {0, 2, 4, 5, 7, 9, 11}.
Predefined minor keys always return: {0, 2, 3, 5, 7, 8, 10}.

Lua-supported (0.58).

Returns
An array (or table in Lua) specifying the chromatic step where each diatonic step occurs.

◆ CalcScaleNumber()

eHarmLev FCKeySignature::CalcScaleNumber ( eHarmLev  displacement) const

Calculate the scale number of the input displacement value.

The scale number is the distance from the tonic, meaning that the number corresponds to moveable solfege syllables as 0-do, 1-re, 2-mi, 3-fa, etc.

This function is useful for tasks like determining the scale degree of an instance of FCNote or for applying the result of an ad hoc transposition to FCChord. If you use FCChord::GetTransposer to get your transposer, you do not have to worry about this. But you might use it, e.g., to transpose the root of a chord while leaving its alternate bass fixed, or vice versa.

Lua-supported (0.67)
