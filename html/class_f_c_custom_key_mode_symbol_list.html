Detailed Description

Class that encapsulates the list of accidental symbols for instances of FCCustomKeyModeDef. A single symbol list may be shared amongst multiple instances of FCCustomKeyModeDef, so these lists must be separately maintained.

See comments at SetList for details on the how the list is stored.

Remarks
Finale supports 15 different key values that correspond to accidentals. They range from -7 to +7 steps including 0 (the natural accidental). When constructing a custom key mode, it is wise to define the accidental symbols (or symbol combinations) for all the 15 available accidentals from -7 to +7 steps. This is because if some accidentals are left undefined, they will not show in the score. If a note were inadvertently to receive an FCNote::GetRaiseLower value that has no accidental character associated with it, the note will look like it has no accidental even though its actual RaiseLower value is something different.

Lua-supported (also as property) (0.72)

Parameters
listtable (Lua) or std::map (C++) of integer keys ranging between -7..+7 (including 0) and utf8-encoded string values that represent the accidental symbols. Not all the key values must be present, but it is preferable to supply them all. (See remarks.)
